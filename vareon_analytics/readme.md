# 📡 Vareon Logging System

The Vareon Logging System is a centralized telemetry and analytics framework used throughout the bot ecosystem.

It records operational events, execution metadata, and long-term activity statistics, providing a consistent audit trail for debugging, monitoring, and reporting.

---

## 📂 Log Categories

The Vareon logging system currently consists of three telemetry layers, each serving a different purpose within the analytics pipeline.

### 1. `live_logs`

Stores every individual event generated by the application and serves as the primary source of telemetry data.

➡️ See: **[live_logs Table](Readme/logging.md)**

---

### 2. `monthly_stats`

Stores monthly aggregated analytics generated from records in `live_logs`.

➡️ See: **[monthly_stats Table](Readme/monthlylog.md)**

---

### 3. `yearly_stats`

Stores long-term analytics generated from monthly statistics.
> Note: The yearly analytics engine is currently under development and may evolve as reporting requirements expand.

➡️ See: **[yearly_stats Table](#-yearly_stats-table)**

---

## 🏗️ High-Level Flow

```text
Application Events
        │
        ▼
    live_logs
        │
        ▼
  monthly_stats
        │
        ▼
   yearly_stats
```
